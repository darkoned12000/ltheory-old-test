# AGENTS.md ‚Äî Limit Theory Old (ltheory-old-test)

Technical reference for AI agents and contributors working on this codebase.
Documents the architecture, third-party libraries, build system, revamp work,
and modernization roadmap. Read before making large changes.

## 1. What This Project Is

- **Limit Theory Old** ‚Äî the original C++ implementation of the *Limit Theory*
  engine (LTE) and the *Limit Theory Scripting Language* (LTSL).
- Authored ~2012‚Äì2015 by Josh Parnell. Forked to
  `github.com/darkoned12000/ltheory-old-test` for modernization.
- The engine is a real-time 3D space game engine: rendering, physics,
  procedural generation, UI, and an embedded scripting language.
- Gameplay is **not** written in C++; it is written in **LTSL** (see ¬ß6) and
  launched by the `launch` executable with an app name, e.g. `war`.

## 2. Repository Layout

```
CMakeLists.txt        Top-level build (project = LTheory)
configure.py          Build/run helper (configure, build, clean, run)
src/launch/           The launcher executable entry point (main())
src/liblt/            The engine library: LTE + all subsystems (built as liblt.so)

script/               Python tooling (tloc, assetlist, install_dependencies.sh)

```

`src/liblt/` subsystem map:
- `LTE/`      ‚Äî engine core: type system, serializer, scripting (LTSL),
               memory, math, reflection (`AutoClass`, `FIELDS`).
- `Game/`     ‚Äî game objects, items, graphics generators, physics hooks.
- `Component/`‚Äî ECS-style components (Drawable, Collidable, Pilotable, ...).
- `UI/`       ‚Äî widgets, glyphs, compositors.
- `Module/`   ‚Äî SoundEngine (FMOD), PhysicsEngine, Scheduler, Script APIs.
- `Audio/`    ‚Äî DSP (envelopes, filters, signal processing).
- `Volume/`   ‚Äî voxel/marching-cubes volume utilities.
- `ThirdParty/` ‚Äî KDTree, triangle overlap, etc.

## 3. Build System

### Toolchain
- **CMake >= 3.10** (raised from 3.0 for CMake 4 compatibility).
- **C++17** (raised from `c++0x`). `-fno-exceptions` kept intentionally
  (the codebase uses no try/catch/throw). Position-independent code required.
- **Compiler**: GCC/Clang on Linux. Originally MSVC on Windows.

### Commands (`configure.py`)
```
python3 configure.py            # cmake configure (-DCMAKE_BUILD_TYPE=RelWithDebInfo)
python3 configure.py build      # cmake --build ./build (parallel, -j $(nproc))
python3 configure.py clean      # rm -rf bin build cache
python3 configure.py run <app>  # runs bin/launch <app> with LD_LIBRARY_PATH set
python3 configure.py test       # runs all unit tests (lte_tests target)
```

### Key CMake facts (top-level `CMakeLists.txt`)
- **SFML 3.1.0** is found via `find_package(SFML 3.1 REQUIRED COMPONENTS ...)`.
  The vendored `ext/SFML/` directory is legacy and not built. CMake imported
  targets (`SFML::Graphics`, etc.) provide include paths and link libraries.
- Linux flags: `-fno-exceptions -O2 -g -msse -msse2 -Wall -Wextra`
  (`-Werror` scoped to project targets `lt` and `launch`; vendored code excluded).
- **Known pre-existing warnings:** GCC 15 emits `-Wunused-parameter` warnings
  from engine template/macro code (`Type.h`, `Common.h`, `Function_Generated.h`,
  `Reference.h`, `AutoClass.h`, etc.) in the `launch` and `lte_tests` targets.
  These come from macro-generated functions (e.g. empty `FIELDS {}` in base types
  producing `MapFields(TypeT*, void*, FieldMapper&, void*)` with all params
  unused). Suppressed by `-Wno-unused-parameter` in `CMAKE_CXX_FLAGS` but some
  template instantiation paths bypass it. **Pre-existing ‚Äî not caused by any
  upgrade.** Low priority; fix at source if/when refactoring the reflection
  macros.
- Link libraries (Linux): `dl freetype`, `SFML::Graphics`, `SFML::Audio`,
  `SFML::Network`, `SFML::System`, `SFML::Window` (CMake imported targets from
  `find_package(SFML 3.1)`), `GLEW`, plus FMOD from `extbin/linux64` via
  `link_directories`.
- `OpenGL` found via `find_package(OpenGL REQUIRED)` with `CMP0072` NEW (GLVND).

### Runtime
- `launch` needs FMOD `.so` files at runtime. `configure.py run` sets
  `LD_LIBRARY_PATH` to `bin/` and `extbin/linux64/`. FMOD `.so` copied into
  `bin/` during setup.
- Run an app: `python3 configure.py run war` (scripts at
  `resource/script/App/<name>.lts`).

## 5. Rendering & Shaders

- Rendering via SFML `sf::RenderWindow` + raw OpenGL (programmable pipeline).
- **Shaders are `.jsl` files** (169 under `resource/shader/`), custom GLSL-ish
  format: `vertex/<name>.jsl` + `fragment/<name>.jsl` pairs.
- Uses **GLSL 3.30 core** syntax. Engine force-prefixes every shader with
  `#version 330 core`. `JSLPreprocess` handles `#include` and `#output` directives.
- OpenGL 3.3+ context (driver serves 4.6 core). Core profile bit intentionally
  left off (SFML 2.6 + Mesa crashes on GLX `MakeCurrent` with Core bit set).
- Draw path is core-compatible: single global VAO bound at init, VBO uploads
  for all geometry (no `glBegin`/immediate mode).
- **Visual Goal (stretch):** PBR transition ‚Äî albedo, normal maps,
  roughness/metallic maps, directional lighting with terminator line.

## 6. LTSL (Limit Theory Scripting Language)

LTSL is a **homegrown scripting language** compiled to an AST and evaluated
via a **Tree-Walking Interpreter**.

- **Architecture:** 25 expression-node types, each evaluated recursively via
  `Evaluate(void*, Environment&)`. Pipeline: tokenize ‚Üí
  `LTSL_ApplyRewrites` (`.` access + infix operator precedence) ‚Üí
  `Expression_Compile` ‚Üí AST.
- **`return` keyword:** Added as Revamp Work (`ExpressionReturn` node). Both
  `return <expr>` and bare `return` are supported. Backward compatible ‚Äî
  last-expression return still works.
- **Benign switch logs:** The `"switch -- case ... did not compile"` messages
  are gated behind `env.detail` (only print during error-replay, not normal
  compilation). They were always benign.
- **C++ bindings:** Functions bound via `Function_Generated.h` macro family
  (`FreeFunctionN`/`MemberFunctionN`/`VoidFreeFunctionN`, ~894 bind sites).
- **Performance:** Tree-walking is sufficient for gameplay logic. Optimize
  C++ `ScriptAPI` bindings before considering a bytecode VM.

Proposed improvements (none started):
- **Data-Driven UI:** Move HUD layout to JSON/XML.
- **Hot-Reloading:** File-watcher to recompile `.lts` at runtime.
- **Component-Based Logic:** Scripts attached to objects (e.g. `Ship` has
  `FlightController` script).

## 7. Revamp Work: `ltheory-main` Universe Sandbox App

A self-contained, seed-driven 3D sandbox app built on top of the engine so
the procedural-universe path can be exercised without touching Josh's original
apps. All files listed here are **Revamp Work** (GPL-3.0).

- **`resource/script/App/ltheory-main.lts`** ‚Äî the app:
  - Implements `Config_Get(key)` loader reading `resource/script/gameConfig.txt`
    (`key:value`, `#` comments, no spaces around `:`). Re-parsed per call ‚Äî
    call from `Initialize()` and cache. (LTSL `String_Split` 2-arg overload is
    not bindable in this build, so parses with `Substring`/`Length`.)
  - Reads `seed`, `loadTime`, `playerCredits`, `shipHull` from config.
  - Creates system via `Object_System (Vec3 15.012) seed` (C++ factory makes
    star + nebula + starfield; does **not** make planets/asteroids).
  - Calls `Object/SystemPopulate:Init root` (which **returns** the planet) to
    add a planet + ~1000-asteroid belt, seeded from `root.GetSeed`.
  - Planet at true-3D `orbitalR * rng.Direction` (random orbital inclination),
    seeded grammar name.
  - Player ship spawned **in the asteroid belt just outside the planet**
    (`planetRadius*2` along `(0.2, 0.6, 1)` direction), rings visible from spawn.
    12 seeded AI ships on `planetRadius*4` shell (outside belt).
  - Third-person camera: `SetRelativePos (Vec3 0 40 160)`.
  - Loading screen (`Custom Widget LoadingScreen`) for `loadTime` seconds,
    then swaps to render pass list: `RenderPass_Clear` ‚Üí `Camera` ‚Üí `SMAA` ‚Üí
    `Interface ui` ‚Üí `PostFilter "post/dither.jsl"`.
  - **Scripted planet bounce:** scans `root.GetInteriorObjects` for
    `o.GetType == "Planet"` and clamps player to `planetRadius * 1.1`.
  - Hotkeys: **F2** = `Widget/DevPanel`, **F3** = `Widget/DebugScene`.

- **`resource/script/Object/SystemPopulate.lts`** ‚Äî populator:
  - `function Object Init (Object self)`; **returns** the planet.
  - Places planet at real orbital radius (not `Vec3_Cylinder 0` like upstream).
  - 1000-rock seeded belt (shell `planetRadius*1.6..2.3`, scale `150*Exp`).
  - Seeded `Grammar_Get "$system"` name. Everything derives from one
    `RNG_MTG(self.GetSeed)` ‚Äî fully deterministic.
  - Does **not** call the WarpNode block (upstream Orbit rail was broken;
    now fixed ‚Äî see appendix A.7).

- **`resource/script/Widget/DebugScene.lts`** ‚Äî debug overlay.
  Lists every interior object of `root` with live XYZ + distance from player,
  colors planet gold, reports nearest-asteroid distance. Toggle: F3.

- **`resource/script/gameConfig.txt`** ‚Äî shared config (`key:value`, `#` comments).
  Keys: `seed`, `loadTime`, `playerCredits`, `shipHull`.

> **LTSL quirks (revamp-work changes noted):**
> - `return` keyword **now works** (added as Revamp Work). Functions still
>   return their last expression for backward compat. See docs/ltsl-docs.md.
> - `switch -- case ... did not compile` lines are **silenced** in normal
>   compilation (gated behind `env.detail`). They were always benign.

> **PlanetType.cpp (Revamp Work):** `atmoDensity` now `0..2` (was frozen at
> `1.0`), `atmoTint` is a seeded hue (was always white), color palette widened
> from narrow reddish range to full hue, and `hasRings` (~60% chance) makes
> rings optional (were unconditional). See ¬ß8.1.

## 8. Universe Generation & Procedural Content Roadmap

**None of this is implemented yet** ‚Äî documented so work can resume without
re-deriving engine facts. Long-term goal: wire the F2 DevTool to live-edit
generator parameters.

### 8.1 Engine facts discovered

- **Planet surfaces have NO biome logic.** Surface generated by
  `PlanetSurface.cpp` ‚Üí `gen/planet.jsl` (4 channels: height, color, clouds,
  spare). Seed-driven variation is noise `coef`/`freq`/`power`/`seed` only.
  Look variety from `Item_PlanetType` (surface tint, `atmoTint`/`atmoDensity`,
  `oceanLevel`/`cloudLevel`). Currently every planet reads as a similar "desert."
- **`Item_PlanetType` was near-identical across seeds** (pre-Revamp).
  Revamp fixed this: atmosphere `0..2`, tint seeded hue, palette full-hue,
  `hasRings` ~60%.
- **Stations exist but are NOT generated.** `Object_Station` +
  `Item_StationType(seed, value, capacity, integrity)` provide full stations
  with docks, market, mission board ‚Äî but nothing spawns them.
- **`Object_System` makes only star + nebula + starfield + dust.**
  No planets, asteroids, or stations.
- **Planet type variation point:** `SystemPopulate.lts` calls
  `Object_Planet (Item_PlanetType (rng.Int + 8))`. The `+8` offset spreads
  seed away from 0.

### 8.2 Config knobs (proposed `gameConfig.txt` keys)

| Key | Meaning | Wiring |
|-----|---------|--------|
| `numOfPlanets` | # planets | loop in `SystemPopulate:Init` |
| `numOfShips` | # AI ships | loop in `ltheory-main` |
| `numOfAsteroids` | # asteroids | loop in `SystemPopulate:Init` |
| `planetRingRatio` | ring probability (0..1) | `Item_PlanetType` arg |
| `dustLevel` | dust fleck density | `Object_DustFlecks` |
| `fogLevel` | scene fog density | `Object_System` / draw state |

| `sunColor` | star tint (RGB) | `GenerateStarColor` path |
| `nebulaIntensity` | nebula opacity/color | `Generator_Nebula_Args` |

### 8.3 Implementation plan

**Pass A ‚Äî config-driven counts + stations + multi-planet** (mostly script,
small C++):
- Wire ¬ß8.2 keys to `ltheory-main.lts` / `SystemPopulate.lts`.
- Loop `numOfPlanets`, `numOfShips`, `numOfAsteroids`.
- Place ~1‚Äì3 stations per system at random seeded positions (not tied to planet).
- Add `ringRatio` arg to `Item_PlanetType`.

**Pass B ‚Äî biomes + visual knobs** (C++/GLSL):

## 9. Engine Modernization: Code Assessment

A code-survey of `src/liblt` (~72k LOC). The engine is in **better shape than
its age implies** ‚Äî already C++17, `nullptr`-dominant, exceptions-off, and
uses programmable shaders. The high-value work is graphics (GLSL) and
correctness/tooling passes, NOT a ground-up rewrite.

### 9.1 State of the code

- **C++ standard:** C++17, `-fno-exceptions`, `-msse -msse2`, `-Werror` scoped
  to project targets.
- **Memory:** ~585 raw `new`/`delete`, but most flow through intrusive refcount
  `Reference<T>` (311 uses) ‚Äî engine's `shared_ptr`. Also `Pointer<T>` (observer)
  and `AutoPtr.h`. **Do NOT** swap `Reference<T>` for `std::shared_ptr` ‚Äî it is
  load-bearing for reflection/serialization.
- **Idioms:** `nullptr` dominant (378 vs 41 `NULL`); `typedef` 539 vs `using` 7;
  ~1094 C-style casts; deliberate `*(T const*)0` type-introspection idioms
  (now centralized in `Type_Ref<T>()`, no longer scattered).
- **Containers:** `Vector.h` wraps `std::vector`; `Array.h`, `Map.h`,
  `VectorMap.h`, `Pool.h` are custom (serializable via `MapFields`).
- **Reflection:** macro system ‚Äî `AutoClass`/`AutoClassDerived` (arities 0..32),
  `FIELDS`/`MAPFIELD`, `AUTOMATIC_REFLECTION_PARAMETRIC1/2`, `METADATA`/`REGISTER_TYPE`.
- **LTSL:** tree-walking interpreter (25 node types, 2,854 LOC). No bytecode/VM/JIT.
- **Graphics:** GLEW + OpenGL 3.3+ context. Programmable pipeline: FBOs, MRT,
  VBOs, single global VAO. Fixed-function (`glBegin`) **removed**.
- **GLSL:** All `.jsl` shaders forced to `#version 330 core`. `JSLPreprocess`
  handles `#include`/`#output`. `texture2D`/`textureCube` ‚Üí overloaded
  `texSample()` wrappers.
- **Audio:** FMOD 4.44 low-level API, **already abstracted** behind
  pure-virtual `SoundEngine` interface (`Fmod` and `Null` implementations,
  selected via `GetSoundEngine()`). New backends possible without touching
  call sites.
- **SFML:** system 3.1.0, uses C++17, scoped enums, miniaudio backend.

### 9.2 Non-goals / Cautions

- **Keep `-fno-exceptions`.** Engine assumes no exceptions; do not introduce
  try/catch/throw.
- **Keep `Reference<T>` / reflection macros.** They are the backbone of
  serialization + scripting.
- **Keep `Shader(vertex, fragment)` LTSL API and `.jsl` layout stable.**
- **Deferred cleanup:** EasyGL fixed-function wrappers in `GL.h`
  (`GL_Begin`/`GL_End`/`GL_Vertex`/etc.) are unused but left in place. Safe
  to delete later once confirmed nothing references them.

### 10.9 Modernization Roadmap (AI Assessed)
To successfully transition this old engine and slowly build a game out of it, the following phased approach is recommended:

**Phase 1: Dependencies & Core Stability (High Priority)**
- [x] **FMOD Replacement:** Strip out the ancient FMOD binaries. Implement a new `SoundEngine` backend using `miniaudio` (a modern, single-file C header library) or leverage SFML's existing `sf::Audio` (OpenAL).
- [x] **SFML 3.x Upgrade:** Move to SFML 3 to gain modern windowing, miniaudio
      audio backend, and C++17/`std::variant` event API. Upgraded through 3.0.2
      ‚Üí 3.1.0 (system packages). **No Wayland backend yet** ‚Äî SFML 3.1 is
      X11-only; runs on Wayland via XWayland.

**Phase 2: Graphics & Rendering Modernization**
- [ ] **OpenGL 4.30 Upgrade:** Bump GLSL versions to 4.30 to unlock compute shaders (useful for volumetric nebulas and fast asteroid instancing).
- [ ] **Physically Based Rendering (PBR):** Update the `.jsl` shaders to support a PBR pipeline (Albedo, Normal, Roughness, Metallic) with directional shadow mapping and atmospheric scattering.

**Phase 3: Gameplay & Procedural Content Tooling**
- [ ] **Live Developer Tools:** Expand `Widget/DevPanel` into a robust debug UI. Connect it so parameters like planet sizes, asteroid density, and biome colors can be tweaked live in-game without restarting.
- [ ] **Data-Driven Configuration:** Move UI layouts and generation seeds into external JSON configs (building off `gameConfig.txt`), separating data from LTSL logic.

## 12. Quick Start (Linux)

```
sudo apt install git-lfs libopenal-dev libvorbis-dev libogg-dev libflac-dev \
                 flac libflac++-dev libglew-dev libfreetype6-dev build-essential cmake
git lfs install
git clone <your-fork-url> ltheory-old-test && cd ltheory-old-test
python3 configure.py
python3 configure.py build
python3 configure.py run war
```

Build uses parallel compilation (`-j $(nproc)`); ~10s on fast hardware after
first full build.

### A.1 Initial CMake 4 / GCC 15 Bring-up Fixes

Resolved during the initial modernization bring-up. Do not reintroduce:

- `GLX` typedef conflicts in `ext/SFML/.../GlxExtensions.hpp` (guarded with
  `#ifndef GLX_VERSION_1_3` etc.).
- `std::auto_ptr` ‚Üí `std::unique_ptr` in `ext/SFML/.../AudioDevice.cpp`.
- `Vector.h` / `Array.h` value-initializing `resize()` (was `resize(sz, T())`).
- `Blueprint.cpp` `Item_##x##_Args* const` ‚Üí `Item_##x##_Args const*`.
- `Serializer.cpp` `resourceID` initialized to `0`.
- `Location.cpp` `free(buf)` ‚Üí `delete[] buf` (was `new char[]`).
- UTF8 `std::iterator` ‚Üí explicit `iterator_category`/`value_type` typedefs.
- **Exit Game button / Escape did nothing.** Fixed by exposing
  `Program_Delete()`/`Exit` (no-arg ScriptAPI free function) and wiring
  `GameMenu.lts` + `Widget/ExitButton.lts` to call it. `Program` now tracks
  current instance via `Program_GetCurrent()`.
- **Button clicks had no effect.** Root cause: `CaptureMouse` on outermost
  wrapper but `PostUpdate()` checked `self.focusMouse` on innermost widget.
  Fixed by adding `CaptureFocus` functions to `Button.lts` / `ExitButton.lts`
  that hit-test via `Mouse_GetHitPos`.
- **Mouse hover/focus tracking broken.** `Mouse_GetPos` returned stale position
  from previous frame. Fixed by using `Mouse_GetPosImmediate()` in `Cursor_Push`.
- **Shutdown crash** (double-free). Root cause: `Launcher` destructor destroyed
  `soundEngine` before `instance`. Fixed by adding `Program::OnDelete()` virtual
  hook called before main loop exits.
- **`Function_Generated.h` missing** (deleted from upstream). Restored from
  `JoshParnell/ltheory-old` GitHub.
- **`Reference<unknown type>` corruption** (type-system SIOF, initial partial
  fix). `RNGT`/`InterfaceT`/`FontT` had no reflection ‚Üí fell through to
  `"unknown type"` fallback with null function pointers. Fixed by adding minimal
  reflection (DeclareMetadata/DefineMetadata) for the three types. Note: they
  are abstract, so `allocate`/`construct` deliberately omitted (only used
  via `Reference<>`). The broader exe/dll cache divergence was later fully
  fixed via cache unification ‚Äî see A.7.

### A.7 Improvement Backlog Items (¬ß8d ‚Äî Completed)

**#1: Fix `Position` engine gap ‚Üí restore warp rails**
- `Position` is the same type as `V3D` (`V3T<double>`), but the script name
  `"Position"` was never bound.
- Fix: dll-side `__attribute__((constructor))` calls
  `Type_AddAlias(Type_Get<V3D>(), "Position")`. Added `SetLook` overload
  (V3D arg) and `ParticleSystem_Add_Position` overload (V3D args).
- **KNOWN TRAP:** Do NOT register `V3D ‚Üí V3F` conversion at static-init or
  call `Type_Get<Position>()` at static-init ‚Äî triggers SIOF that corrupts
  the global type/function registry (numeric literals fail). Do NOT add
  mixed-precision operator overloads ‚Äî creates ambiguous resolution for ship
  physics (`force.IsFinite()` assertion ‚Üí NaN ‚Üí SIGABRT).
- Verification: `rails` 3/3 clean runs; `ltheory-main`/`war` compile
  WarpNode/WarpRail with no errors; `configure.py test` 60 checks, 0 failures.

**#2: Fix `Thread_Create` / `GetResult`** ‚Äî See A.5 (Content/LTSL).

**#9: CapsLock / ESC keyboard-LED quirk on XWayland** (investigated,
documented ‚Äî not an engine bug)
- Pressing ESC on Wayland + XWayland causes CapsLock LED to turn on and stay
  on after exit. Root cause is NOT in the engine ‚Äî ESC ‚Üí Exit ‚Üí Program::Delete
  never touches X11/XKB/LED state. The flip is an XWayland keymap/XKB desync
  artifact.
- Mitigation: tap CapsLock to clear LED; or run pure X11 session.
- No engine change warranted. Data point for SFML 3/Wayland upgrade decision
  (native Wayland removes XWayland layer where quirk lives).
